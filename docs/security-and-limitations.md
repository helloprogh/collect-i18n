# 安全与局限

## 本地信任边界

Collect I18n 只应对本地、可信且已获授权的项目运行。工作台/API 只绑定 `127.0.0.1`，每次服务启动都会生成随机 capability：CLI 使用 Bearer 凭据访问 API，工作台通过带 capability 的 `/auth` 地址换取 `HttpOnly`、`SameSite=Strict` Cookie 后立即重定向到无凭据 URL。API 不发送 CORS 许可，并拒绝携带工作台 Cookie 的跨源请求。

`start` 返回的 `studioUrl` 含当前服务的 capability，应按临时访问凭据保护，不要发布到不受信任的聊天、工单或日志。即使服务只监听回环地址，也不要端口转发、反向代理或把它暴露到局域网/公网；能够读取项目 `.collect-i18n/service.json` 的其他本机进程仍处于同一信任边界内。

采集器会加载并执行目标项目的 Vite 配置、插件和前端代码。这等价于在本机运行该项目的开发服务器，不能用它检查来源不明的仓库。

## 敏感数据

目标项目的 `.collect-i18n/` 可能包含：

- Chrome profile 中的 Cookie、Local Storage 和登录态；
- `config.json` 中配置的浏览器 Cookie、`service.json` 中的 capability 以及尚未完成重定向的工作台认证 URL；
- 页面截图中的姓名、账号、业务数据或访问令牌；
- SQLite 中的中文文案、源码位置、路由和错误；
- TriggerPlan、请求 Mock 响应和服务日志；
- 导入、导出的翻译工作簿。

把 `.collect-i18n/` 加入目标项目 `.gitignore`，不要提交或上传整个目录。共享截图/Excel 前先人工检查敏感内容。任务结束后先运行 `stop`，再按组织的数据保留规则清理 profile、导入文件和截图。

TriggerPlan 不允许读取环境变量。`fill.value` 只用于无敏感性的测试数据；不要在 Mock body、计划、截图场景或 CLI 参数中放真实密钥。

## 请求 Mock

Mock 由采集器的 Playwright 页面路由实现，只影响该采集浏览器，不修改目标项目代码或系统网络。仍需遵守以下约束：

- 只为源码明确存在的成功/失败状态构造最小响应；
- 尽量限定 HTTP 方法和 URL，避免宽泛的 `**/*`；
- 优先使用 `once: true`，避免后续任务继承不必要的行为；
- 不凭空创造 UI 文案；响应字段和值应有 API 客户端、类型或 fixture 依据；
- 涉及真实写操作时，用 Mock 或测试环境，禁止在生产后端上采集。

## Excel 写回

导入器会校验四列表头、Key Path、中文原文和目标目录，并用真实路径检查防止 `en-us` 越界。已有 JSON 以 `.bak` 备份，写入通过临时文件替换，且尽量保留 BOM、缩进、换行和尾换行。

这些保护不能替代版本控制：

1. 在干净工作区运行 `--dry-run`；
2. 审阅 `issues` 与 `changes`；
3. 确认文件映射后再显式运行 `--apply`；
4. 使用 Git diff 审核所有语言包变化，再删除备份。

不要从不可信来源导入工作簿。上传接口限制为 100 MiB，但大型或恶意 XLSX 仍可能消耗大量内存/CPU。

## 已知局限

- 仅静态字符串 key 能获得最完整的源码关联；运行时拼接、远程下发或计算生成的 key 可能进入未知项或需要人工处理。
- 路由识别依赖常见 Vue Router 配置、静态 import 和导航调用。动态路由、微前端、iframe、跨域页面及复杂重定向可能无法自动抵达。
- Shadow DOM、Canvas、WebGL、虚拟滚动中未挂载的行、极短暂动画和不可见节点不保证取得稳定矩形。
- Element Plus 常见组件属性和命令式服务已覆盖；高度封装的二次组件、其他 UI 库或自定义渲染器可能需要新的适配器。
- 登录、MFA、验证码、文件上传、硬件交互、生产数据依赖和多用户协作属于人工兜底边界。
- 静态阶段按可靠路由批量执行，Agent 阶段顺序执行。每词条一分钟是优化目标，不是 SLA；正确证据优先于吞吐。
- 页面必须在采集窗口内真实渲染目标文案。代码里存在 key、网络响应成功或 Agent 看到相似中文都不等于采集成功。
- 一个目标项目当前只应运行一个后台服务；并发会话、远程队列和团队服务器不是 0.1.0 的目标。

## 生产隔离

采集插件设计用于开发服务器，默认由 Collect I18n 启动器临时追加。不要把 collector runtime、调试 overlay 或 `.collect-i18n` 产物打进生产应用。请求 Mock、测试账号和环境变量应使用隔离的测试环境，不应指向真实生产系统。
